" 강남구 식당 API를 가져와서
데일리 식사메뉴를 랜덤으로 추천하는
프로그램입니다. "

구현목표

  • 다양한 메뉴를 랜덤하게 추천한다.
  • 카테고리 추천과 계절에 따른 메뉴추천을 구현한다.
  • 사용자의 매장 및 음식 평가 열람 기능을 제공한다

기능구현

  • 관리자 : 카테고리 추가 및 삭제 기능
  • 관리자 : 사용자의 후기 내역 조회 및 삭제 기능
          (조회 기준 : 별점순, 매장명 검색, 회원번호 검색)
  • 더미 데이터 생성

페이지 상세설명


관리자 파트의 '카테고리 관리' 파트입니다.

카테고리는 추가 기능과 삭제 기능 중에 선택할 수 있습니다.
관리자 파트의 '카테고리 추가' 파트입니다.

기존 리스트를 확인 후, 번호와 카테고리명을
추가할 수 있습니다.
번호와 카테고리 둘 중 하나라도 쓰지 않으면 안되고,
둘 중 하나라도 중복이 존재하면 추가할 수 없습니다.
관리자 파트의 '카테고리 삭제' 파트입니다.

기존 리스트를 확인 후, 번호로 카테고리 삭제가 가능합니다.
번호로만 삭제가 가능하며 존재하지 않는 번호 또는
카테고리명으로는 삭제가 불가능합니다.
삭제를 하기전에 한번 더 확인하는 질문을 거친 후에
삭제합니다.
관리자 파트의 '매장 및 음식 후기관리' 파트입니다.

사용자가 남긴 후기들을 조회하거나 삭제할 수 있습니다.
조회는 '별점 낮은순', '별점 높은순', '매장명 검색',
'회원번호 검색' 중에 선택하여 조회하실 수 있습니다.
관리자 파트의 '후기 조회 중 별점순' 파트입니다.

별점 높은 순으로 조회를 선택하면 평점이 높은순으로
정렬이 되어 출력됩니다.
선택지에 없는 번호를 선택하면 다시 선택하라는
안내 메세지가 뜨게 됩니다.
0번을 입력하면 다시 후기조회 또는 삭제를 선택하는
파트로 돌아갑니다.
관리자 파트의 '후기 조회 중 매장명 검색' 파트입니다.

검색어를 입력하면 해당 검색어가 포함되어 있는 매장명이 전부 출력됩니다.
정렬은 평점이 높은순으로 정렬되어 출력됩니다.
관리자 파트의 '후기 조회 중 회원번호 검색' 파트입니다.

확인하고 싶은 회원의 번호를 검색하면 해당 회원이
남긴 후기를 조회할 수 있습니다.
일치하는 회원번호가 없다면 재검색을 하도록 합니다.
관리자 파트의 '후기 삭제' 파트입니다.

먼저 회원번호를 검색 후 회원이 남긴 후기를 확인합니다.
회원이 남긴 여러 후기들 중 삭제할 후기의 ID를 입력하면 확인 메세지를 통해 한번 더 확인 후 삭제합니다.
회원번호 또는 후기ID가 일치하지 않는다면
재검색을 하도록 합니다.
